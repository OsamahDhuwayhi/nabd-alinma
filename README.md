# 🩺 نبض الإنماء — رادار صوت العميل (Nabd Alinma)

أداة GenAI لخدمة العملاء تسحب تقييمات تطبيقات **مصرف الإنماء** العامة من Google Play،
تصنّفها إلى **مواضيع ألم**، ترتّبها حسب **الأولوية**، وتقترح **ردودًا متوافقة مع الشريعة**
لفريق تجربة العميل — **كل ذلك مجاني 100%** (لا واجهات مدفوعة، لا استضافة مدفوعة).

> مشروع لهاكاثون **أَمَد** (أكاديمية طويق × مصرف الإنماء) — مسار تجربة العميل.

---

## 💡 الفكرة الأساسية
مصرف الإنماء يشغّل 4 تطبيقات فيها آلاف التقييمات العامة بالعربية والإنجليزية.
دراسة محكّمة سنة 2023 (MDPI Sustainability) وثّقت أن الردود على تقييمات التطبيقات كانت **0%**.

**ماذا تقول بياناتنا الحيّة (آخر سحب فعلي)؟**

| التطبيق | عدد التقييمات | نسبة ردود المصرف |
|---|---|---|
| Alinma App (الرئيسي) | 600 | **60.7%** |
| Alinma Capital (الاستثمار) | 399 | 22.1% |
| **AlinmaPay (المحفظة)** | 600 | **0.0%** 🔴 |
| **Alinma Business** | 67 | **0.0%** 🔴 |
| **الإجمالي** | **1,666** | **27.1%** |

- تحسّن المصرف على تطبيقه الرئيسي، **لكن AlinmaPay و Alinma Business ما زالا عند 0%**.
- يوجد الآن **381 تقييمًا سلبيًا (نجمة أو نجمتين) بلا أي رد** — وهذا ما تستهدفه أداتنا.
- متوسط التقييم العام: **3.72 / 5**.

> ⚠️ هذه أرقام حيّة تتغيّر مع كل عملية سحب جديدة — لا تُختلق أي إحصائية.

## 🧩 المكوّنات
| الملف | الوظيفة |
|---|---|
| `1_scrape_reviews.py` | سحب ~300 تقييم لكل تطبيق ولكل لغة (عربي/إنجليزي) من السوق السعودي |
| `2_categorize.py` | تصنيف المواضيع + حساب درجة الأولوية (مع ترقية ML اختيارية) |
| `3_generate_reply.py` | عرض توضيحي لتوليد الردود (يستدعي `reply_engine.py`) |
| `reply_engine.py` | محرّك الردود ذو الطبقات الثلاث (Groq → Gemini → قوالب) |
| `app.py` | لوحة Streamlit التفاعلية (مركز العرض) |

## ⚙️ التثبيت (مجاني بالكامل)
يتطلب **Python 3.9+**.

```bash
pip install -r requirements.txt
```

## ▶️ ترتيب التشغيل
```bash
# 1) سحب التقييمات الحقيقية  ->  data/alinma_reviews_raw.csv
python 1_scrape_reviews.py

# 2) التصنيف والأولوية       ->  data/alinma_reviews_categorized.csv
python 2_categorize.py
#    (اختياري) ترقية دلالية مجانية بالتعلّم الآلي:
#    python 2_categorize.py --ml

# 3) تجربة توليد الردود (اختياري للاختبار)
python 3_generate_reply.py

# 4) تشغيل اللوحة (مركز العرض)
streamlit run app.py
```

## 🔑 مفتاح Groq المجاني (اختياري — لترقية الردود بالذكاء الاصطناعي)
اللوحة تعمل **بدون أي مفتاح** عبر القوالب الجاهزة. ولتفعيل ردود الذكاء الاصطناعي مجانًا:

1. ادخل على <https://console.groq.com> وسجّل مجانًا.
2. من صفحة **API Keys** أنشئ مفتاحًا جديدًا (يبدأ بـ `gsk_...`).
3. اضبطه كمتغيّر بيئة قبل تشغيل اللوحة:

**PowerShell (ويندوز):**
```powershell
$env:GROQ_API_KEY = "gsk_ضع_مفتاحك_هنا"
streamlit run app.py
```
**Linux / Mac:**
```bash
export GROQ_API_KEY="gsk_ضع_مفتاحك_هنا"
streamlit run app.py
```

> بديل: يمكن استخدام `GEMINI_API_KEY` من Google AI Studio (مجاني) بنفس الطريقة.

## 📸 قائمة اللقطات لإثبات الـ 30% (Screenshot Checklist)
التقط هذه اللقطات لشريحة "الإنجاز 30%":

1. **مخرجات السحب في الطرفية** (`python 1_scrape_reviews.py`)
   → يُثبت أننا نسحب **1,666 تقييمًا حقيقيًا** ونحسب نسبة الردود.
2. **ملف `alinma_reviews_categorized.csv`** مفتوحًا في Excel
   → يُثبت وجود أعمدة `category` و`priority_score` على بيانات حقيقية.
3. **بطاقات المؤشرات في اللوحة** (KPI) مع تنبيه "AlinmaPay و Business = 0%"
   → يُثبت المشكلة الحقيقية بالأرقام.
4. **طابور الأولوية + زر "توليد رد مقترح"** وقد ظهر رد عربي مهذّب
   → يُثبت توليد رد متوافق مع الشريعة (مجانًا).
5. **رسالة "تم اعتماد الرد وحفظه"** بعد الضغط على "اعتماد وتصدير"
   → يُثبت دورة العمل الكاملة مع بقاء الإنسان في الحلقة (human-in-loop).
